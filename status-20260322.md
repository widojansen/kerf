## What Was Built

### 1. vLLM Deployment on DGX Spark

**Deployed:**
- NGC container `nvcr.io/nvidia/vllm:26.02-py3` (14.7GB)
- Model: `nvidia/Qwen3-32B-NVFP4` (NVFP4 quantization, native Blackwell FP4 tensor cores)
- Flags: `--gpu-memory-utilization 0.7 --enable-auto-tool-choice --tool-call-parser hermes`
- KV cache: 496K tokens, 12x concurrency at 40K context length
- Installed as systemd service: `sudo systemctl enable vllm`

**ExClaw VLLMProvider:**
- `lib/exclaw/llm/vllm_provider.ex` — GenServer speaking OpenAI `/v1/chat/completions`
- Anthropic → OpenAI tool format conversion (transparent to callers)
- Anthropic → OpenAI message format conversion (tool_use → tool_calls, tool_result → tool role)
- Empty `tool_calls: []` handled as text response (vLLM quirk)
- 120s `receive_timeout` for thinking models
- Tool call response uses `:input` key matching Session contract
- 8 unit tests

**Infrastructure updates:**
- `lib/exclaw/llm/supervisor.ex` — supports `LLM_BACKEND=vllm|ollama|unset`
- `lib/exclaw/llm/model_router.ex` — 3-provider dispatch (vLLM, Ollama, Anthropic)
- `lib/exclaw/llm/provider.ex` — timeout increased to 180s
- `lib/exclaw/agent/session.ex` — timeout increased to 180s
- `config/runtime.exs` — `VLLM_URL`, `VLLM_MODEL` env vars

### 3. SQLite → PostgreSQL Migration

**Why:** pgvector for vector search over conversation history, AGE for knowledge graphs, concurrent access from multiple services, production-grade database.

**What changed:**
- `mix.exs` — replaced `ecto_sqlite3` with `ecto_sql` + `postgrex` + `pgvector`
- `lib/exclaw/repo.ex` — adapter changed to `Ecto.Adapters.Postgres`
- `lib/exclaw/postgrex_types.ex` — Postgrex types with pgvector extensions
- `config/config.exs` — added `types: ExClaw.PostgrexTypes`
- `config/dev.exs` — PostgreSQL config (database: exclaw_dev)
- `config/test.exs` — PostgreSQL config (database: exclaw_test, SQL Sandbox)
- `config/prod.exs` — PostgreSQL config (database: exclaw_prod)
- `lib/exclaw/memory/store.ex` — `LIKE` → `ILIKE` for case-insensitive search
- SQLite dependency and all `.db` files removed entirely

**New migrations:**
- `20260322120000_add_pg_extensions.exs` — `CREATE EXTENSION vector` + `CREATE EXTENSION age`
- `20260322120100_add_vector_columns.exs` — embedding columns (vector 1024) on messages and memories, HNSW indexes for approximate nearest neighbor search

**Databases created on Spark:**
- `exclaw_dev` — development
- `exclaw_test` — test suite
- `exclaw_prod` — production (Tina via systemd)
